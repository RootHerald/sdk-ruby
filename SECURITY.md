# Security Policy

## Reporting a vulnerability

If you find a security issue in this repository, **please do not open a public issue or PR**. Instead, report it privately so we can investigate and fix it before disclosure.

**Email:** security@rootherald.io

Please include:
- A description of the vulnerability and the impact
- Steps to reproduce (or a minimal proof of concept)
- Affected versions, OS, and configuration
- Your contact information so we can follow up

We will acknowledge your report within 72 hours and provide a timeline for a fix.

## Supported versions

Each Root Herald repo maintains its own version range in its release notes. We support fixes on the latest minor release line of each major version.

## Disclosure

We follow a coordinated disclosure model. Once a fix is available, we will:
1. Release the fix in a patched version
2. Publish a security advisory on GitHub with CVE assignment where appropriate
3. Credit the reporter (if they wish to be credited)

We do not pay bounties at this time but are happy to publicly credit researchers.
